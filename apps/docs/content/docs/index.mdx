## What is Inscribe?

**Inscribe** is an open-source, canvas-rendered document editor framework. Unlike traditional
DOM-based rich-text editors, Inscribe renders its content directly onto `<canvas>` elements ŌĆö
giving you pixel-perfect, paginated layouts with high performance virtual scrolling, without
fighting the browser's layout engine.

It is built on the shoulders of giants:

| Layer | Technology | Purpose |
| --- | --- | --- |
| **Document model** | [ProseMirror](https://prosemirror.net/) | Structured document state, commands, and history |
| **Rendering** | HTML5 Canvas | Pixel-perfect, paginated page layout |
| **Collaboration** | [Yjs](https://yjs.dev/) + [Hocuspocus](https://hocuspocus.dev/) | Real-time CRDT-based co-editing |
| **React bindings** | `@inscribe/react` | Hooks and components for mounting the editor in a React app |

## Key Concepts at a Glance

**Canvas rendering, not DOM.** Every glyph, cursor, selection highlight, and overlay (ghost text,
tracked changes) is drawn directly on an `HTMLCanvasElement`. The editor owns its own coordinate
system ŌĆö the browser's layout engine is only used for the outer scroll container.

**Pages as first-class citizens.** The document is broken into fixed-size pages (A4 by default).
The `ViewManager` implements virtual scrolling, only painting pages that are currently in the
viewport. You control page dimensions and margins through a simple `PageConfig` object.

**Extension-first architecture.** Almost every feature ŌĆö headings, lists, links, collaboration ŌĆö
is an `Extension`. You compose the features you need via `StarterKit` or by listing extensions
explicitly. This keeps bundle sizes lean.

**One document model + multiple renderers.** The ProseMirror `EditorState` is the single source of
truth. The canvas renderer reads from it; Yjs syncs it across peers; the export layer serializes it.
Nothing holds a shadow copy.
