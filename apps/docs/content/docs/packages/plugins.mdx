`@inscribe/plugins` ships optional extensions that build on top of `@inscribe/core`. Install it only
when you need one of the advanced features it provides.

## Installation

    ```

    ```
  </Tab>
  <Tab value="yarn">

## Real-Time Collaboration

Inscribe's collaboration layer uses **Yjs** for CRDT data structures and syncs over WebSockets via
**Hocuspocus**. Two extensions are needed:

- `Collaboration` — binds the ProseMirror document to a shared Yjs `Y.Doc`
- `CollaborationCursor` — renders remote user cursors on the canvas overlay

### Setup

First, install the peer dependencies:

<Tabs items={["pnpm", "npm"]}>
  <Tab value="pnpm">
    ```bash
    pnpm add @hocuspocus/provider yjs
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    npm install @hocuspocus/provider yjs
    ```
  </Tab>
</Tabs>

Then configure your editor:

```tsx title="CollaborativeEditor.tsx"
import { useCanvasEditor, Canvas, StarterKit } from '@inscribe/react';
import { Collaboration, CollaborationCursor } from '@inscribe/plugins';
import { HocuspocusProvider } from '@hocuspocus/provider';

const provider = new HocuspocusProvider({
  url: import.meta.env.VITE_WS_URL,
  name: 'my-document',
});

export function CollaborativeEditor() {
  const editor = useCanvasEditor({
    extensions: [
      StarterKit,
      Collaboration.configure({ document: provider.document }),
      CollaborationCursor.configure({
        provider,
        user: { name: 'Alice', color: '#f7b731' },
      }),
    ],
  });

  return <Canvas editor={editor} style={{ height: '100vh' }} />;
}
```

**Environment variable required:** `VITE_WS_URL` — the WebSocket URL of your Hocuspocus server
(e.g. `ws://localhost:1234`).

See the [Collaboration example](/ examples/collaboration) for a full backend + frontend setup.

## AI Toolkit

The AI Toolkit adds a streaming ghost-text writing assistant powered by any LLM via the
**Vercel AI SDK**. It renders suggestions as a canvas overlay — the document is never mutated until
the user explicitly accepts a suggestion.

### Setup

Install the AI SDK peer dependencies:

<Tabs items={["pnpm", "npm"]}>
  <Tab value="pnpm">
    ```bash
    pnpm add ai @ai-sdk/react @ai-sdk/anthropic
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    npm install ai @ai-sdk/react @ai-sdk/anthropic
    ```
  </Tab>
</Tabs>

**Environment variable required:** `ANTHROPIC_API_KEY` — set on your backend API route.

```tsx title="AiEditor.tsx"
import { useCanvasEditor, Canvas, StarterKit } from '@inscribe/react';
import { AiToolkit } from '@inscribe/plugins';

export function AiEditor() {
  const editor = useCanvasEditor({
    extensions: [
      StarterKit,
      AiToolkit.configure({
        endpoint: '/api/ai/complete', // your streaming API route
      }),
    ],
  });

  return <Canvas editor={editor} style={{ height: '100vh' }} />;
}
```

See the [AI Assistant example](/ examples/ai-assistant) for the full API route implementation.

## Track Changes

The Track Changes extension records every insertion and deletion as annotated ranges in the document,
enabling multi-author review workflows.

```tsx
import { TrackChanges } from '@inscribe/plugins';

const editor = useCanvasEditor({
  extensions: [
    StarterKit,
    TrackChanges.configure({
      enabled: true,
      author: { id: 'user-1', name: 'Alice' },
    }),
  ],
});

// Accept all pending changes
editor.commands.acceptAllChanges();

// Reject a specific change
editor.commands.rejectChange(changeId);
```

<Callout type="info">
  Track Changes stores annotations directly in the ProseMirror document as marks. It is fully
  compatible with the Collaboration extension — multiple authors' changes are tracked independently.
</Callout>
